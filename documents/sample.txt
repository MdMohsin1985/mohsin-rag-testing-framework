Retrieval augmented generation, often called RAG, is a pattern for answering
questions with help from external documents. A RAG system first retrieves
relevant passages from a knowledge base and then asks a language model to
answer using those passages as context.

This sample document belongs to the AI RAG Testing Framework. It explains that
documents are loaded from the documents folder, split into chunks, stored in
ChromaDB, retrieved by semantic similarity, and passed to OpenAI for answer
generation.

Evaluation helps verify that the generated answer is useful and grounded in
the retrieved context. Faithfulness checks whether the answer is supported by
the context. Answer relevancy checks whether the answer directly addresses the
user question.
